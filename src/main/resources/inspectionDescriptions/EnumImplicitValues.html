Reports TypeScript enums whose members do not have explicit values assigned.

When enum members have no explicit values, TypeScript assigns sequential numbers starting from zero. If a new member is added or existing members are reordered, all values shift silently. This can cause runtime inconsistencies when values are persisted (e.g. in localStorage) or transmitted over a network.

Assign explicit string or number literals to each enum member to prevent unexpected behavior from reordering.

Applies to: <script setup lang="ts"> blocks only.
