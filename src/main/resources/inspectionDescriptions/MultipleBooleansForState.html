Reports components that define too many boolean ref() values to manage state (e.g. isLoading, isRunning, isPaused).

Multiple boolean states make it easy to accidentally set contradictory combinations (e.g. both isRunning and isFinished set to true). TypeScript cannot enforce valid state transitions with separate booleans.

Use an enum or union type to restrict the component to one state at a time, ensuring only valid transitions are possible.

Default threshold: 4 boolean refs (based on Nunes et al., 2025).
